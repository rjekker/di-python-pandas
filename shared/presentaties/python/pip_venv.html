Installing Libraries

Virtual Environments

A virtual environment is an isolated environment for a project

You can install packages in here (this is a different kind of package!)

This will not affect the rest of your system

Creating a venv

python -m venv name-of-env

This will create a new folder called name-of-venv

This has its own python, installed packages, and more

Pycharm

Pycharm automatically creates a venv for each project

Venv activation

Always work inside an active virtual environment

Run the activate script inside the environment: 

# on linux/mac
. env-name/bin/activate

# on windows
env-name\Scripts\activate.bat

You will see the command prompt change: the name of the environment will be shown at the start, in parens.

Installing packages

Use Pip

Pip is a part of your python installation 

python -m pip install requests

Other commands

python -m pip uninstall requests
python -m pip show requests
python -m pip list

Anaconda

Anaconda includes the command conda which offers functionality from both pip and venv.
