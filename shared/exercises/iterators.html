Deck of cards

Consider the following code: 

from collections import namedtuple

card = namedtuple('Card', ['rank', 'suit'])

class DeckOfCards:
    ranks = list(range(2, 11)) + list('JQKA')
    suits = 'clubs diamonds hearts spades'.split()

    def __init__(self):
        self._cards = [card(rank, suit) for suit in self.suits for rank in self.ranks]

    # Add your own code here

deck = DeckOfCards()
for card in deck:
    print(card)

Currently this programs does not work. Implement iteration over the class to fix this.

Shuffle cards

Implement a generator method shuffle() in the DeckOfCards class that returns a shuffled list of cards, using random.shuffle() (see here).

You would use the generator like this: 

deck = DeckOfCards()
for card in deck.shuffle():
    print(card)

# Let's show that original list is unshuffled
for card in deck:
    print(card)

Note: make sure that the original list self._cards does NOT get shuffled!

A movie class

Create a class Movie which has at least:

  • an id
  • an attribute url which is constructed based on the id
  • a lazy property data. When this is used, you retrieve the data for the movie from the REST Api.
  • a property title. This looks up the title in data so that asking for the title will cause the data to be retrieved.
import requests


class Movie:
    BASE_URL = "https://swapi.dev/api/films"

m = Movie(3)
print(m.title) # causes the data to be loaded
# output: Return of the Jedi

Movie Characters

Add a generator method called characters() to your Movie class, this should use the generator method "characters" to retrieve the characters in a lazy way.

This allows us to do the following: 

m = Movie(1) # passing the id
for character in m.characters:
    print(character)

There should be a REST call for each character only at the moment that we get it from the generator.
