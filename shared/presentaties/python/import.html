Modules, packages, import

import

The import statement lets us use names from other Python modules (.py files) 

from math import sqrt
print(sqrt(5))

2.23606797749979
  • This can be anything: variables, functions, classes, etc.

import

I can import an entire module or just a single name 

import math
print(math.sqrt(5))

from math import sqrt
print(sqrt(5))

Best practices

The wildcard import is frowned upon 

from something import *

Preferably we import just the things we need 

from something import bread, cheese, butter

Import as

Re-name an import for local use. Mostly used for abbreviating long names. 

import matplotlib.pyplot as plt

plt.figure()

Locating files

Where does Python search for files to import?

  1. Current directory (same directory as script doing the importing).
  2. Standard library.
  3. The paths defined in sys.path

Sys.path

A variable available in python: 

import sys

print(sys.path)

['', '/usr/lib/python37.zip', '/usr/lib/python3.7', ...]

Imports and main

What happens when we import a module?

Can this have side effects?

Main

When we write a Python file that can be run as a script,

The code to start the program should use the following if-statement: 

if __name__ == '__main__':
    print("Let's do stuff!")

This make sure that you can import the file without side effects.

Packages

Package

A package is a directory containing Python files

Must always contain a file __init__.py (can be empty).

  • Note: don't confuse this with a package you can download and install.
