Definitie: Internet

Een wereldwijd computernetwerk dat gebruik maakt van (voornamelijk) TCP/IP.

  • Oorsprong: ARPANET in de jaren '60.
  • Computers en deelnetwerken aan- en afkoppelen
  • Protocollen zijn geheel open (!)

Internet: Organisaties

  • IETF (Internet Engineering Task Force): IPv4, IPv6
  • ICANN (Internet Corporation for Assigned Names and Numbers): adressen

Definitie: WWW

An open source information space where documents and other web resources

  • are identified by URLs,
  • interlinked by hypertext links,
  • and can be accessed via the Internet.
  • … en via HTTP geserveerd natuurlijk!

WWW

  • Oorsprong: Tim Berners-Lee in 1989,
  • eerste browser: 1990 at CERN, Zwitserland

Organisaties:

  • World Wide Web Consortium (W3C): HTML, CSS, DOM
  • Ecma International: EcmaScript

Protocollen

OSI Model

Het OSI Model

Als applicatie-programmeurs zijn wij voornamelijk gericht op de bovenste drie lagen van het model

Zie ook Internet Protocol Suite

IP

Internet Protocol

  • Verstuurt packages van het ene IP-adres naar het andere IP-adres
  • Geen garantie dat data intact aankomt, of in de juiste volgorde
  • Twee versies: IPv4 en IPv6
  • Voor vertalen van domein-namen naar adressen: DNS

TCP

Transmission Control Protocol

  • Laag bovenop IP
  • Voegt het concept van een connectie toe
  • Zorgt dat data correct aankomt, packages worden in de juiste volgorde gezet
  • De endpoints van de connecties heten sockets en deze hebben een port number

TCP: servers

Een server applicatie "luistert" naar connecties op een specifieke poort

Standaard poorten:

  • HTTP: 80
  • SMTP: 25
  • SSH: 22
  • HTTPS: 443

UDP

Een alternatief voor TCP, voor connecties waar performance belangrijker is dan correctheid (video/audio bijvoorbeeld)

HTTP: HyperText Transfer Protocol

Wat is HTTP

  • Text-based server/client protocol
  • Voor versturen van allerlei soorten data
  • Serveren van web applicaties (HTML/CSS/JS) naar een browser
  • Vele andere soorten toepassingen

Default poort: 80

Wat je nooit mag vergeten

De HTTP cyclus:

  1. De Client stuurt een Request naar de Server
  2. De Server stuurt een Response terug.

En niet anders!

Ook belangrijk

Request en Response zijn tekst.

HTTP is Stateless: geen "geheugen" tussen verschillende request/response cycles.

Zie ook Deze uitleg van het begrip statelessness

HTTP/2

Een sneller, beter (Of niet?) HTTP-protocol. Geen tekst maar binair! HTTP/2 op wikipedia

URLs

Anatomie van een URL

http://www.myserver.org:100/something?x=10

http protocol (http, https, ftp, …)
:// delimiter
www.myserver.org domain name van server
:100 poort van server. Weglaten als default (http -> 80)
/something?x=10 path gedeelte van URL

Het path bepaalt welke resource ik verwacht terug te krijgen van de server.

Wat gebeurt er..?

Wat gebeurt er in de browser als ik deze URL in de adresbalk invoer?

http://www.myserver.org:100/something?x=10

  1. DNS lookup www.myserver.org -> IP Adres
  2. TCP connectie met server op poort 100
  3. HTTP GET Request voor "/something?x=10"
  4. Of click hier als je het precies wilt weten

HTTP Request

Anatomie

The request message consists of the following:

  • A request line: Method and Resource
  • Request header fields (e.g., Accept-Language: en).
  • An empty line.
  • An optional message body.

Simpel voorbeeld

GET /index.html HTTP/1.1
Host: www.google.com

telnet als TCP client

Request Line

De Request Line is de eerste regel van de request

  • Method: GET, POST, PUT, DELETE, HEAD, etc.
  • Resource: het "path" gedeelte van URL
  • eindigt met "HTTP/1.1"

Headers

  • Key/value pairs
  • Syntax: Key: Value
  • Alles mag
  • Eindigt met lege regel

Request body

Voor meesturen van data bij een POST (bijv file upload, form data)

Minder simpel voorbeeld

POST /bestel/form.html HTTP/1.0
User-Agent: SomeBrowser 5.8
Content-Type: application/x-www-form-urlencoded
Content-Length: 44

naam=reindert&product=OCA_Boek&hoeveelheid=1

HTTP Response

Response Anatomie

Lijkt erg op een Request.

  • Statuscode
  • Headers
  • Empty Line
  • Response body

Voorbeeld

HTTP/1.1 200 OK
Date: Mon, 27 Jul 2009 12:28:53 GMT
Server: Apache/2.2.14 (Win32)
Last-Modified: Wed, 22 Jul 2009 19:15:56 GMT
Content-Length: 88
Content-Type: text/html
Connection: Closed

<html>
<body>
<h1>Hello, World!</h1>
</body>
</html>

Statuscodes

  • 1xx Informational
  • 2xx Success
  • 3xx Redirection
  • 4xx Client Error
  • 5xx Server Error
