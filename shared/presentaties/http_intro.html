Http Protocol

  • Text-based server/client protocol
  • Send any kind of data
  • Serve web sites/apps (HTML/CSS/JS) to browser
  • Serve REST data to apps and other programs
  • … and many other uses

Default poort: 80

The cycle

The HTTP cycle:

  1. The Client sends a Request to the Server
  2. The Server sends a Response back.

And nothing else!

Some other things

Request and Response are text messages.

HTTP is Stateless: there is no "memory" between different request/response cycles.

Statelessness explained

HTTP: HyperText Transfer Protocol

URLs

Anatomy of a URL

http://www.myserver.org:100/something?x=10

http protocol (http, https, ftp, …)
:// delimiter
www.myserver.org domain name of server
:100 Server port. Omit for default (http -> 80)
/something?x=10 path part of URL

The path specifies which resource I am expecting to receive from the server.

HTTP Request

Anatomy

The request message consists of the following:

  • A request line: Method and Resource
  • Request header fields (e.g., Accept-Language: en).
  • An empty line.
  • An optional message body.

Minimal Example

GET /index.html HTTP/1.1
Host: www.google.com

Demo: A Human Browser

telnet as TCP client

Request Line

De Request Line is the first line in the reques

  • Method: GET, POST, PUT, DELETE, HEAD, etc.
  • Resource: "path" part of URL
  • ends with HTTP version - "HTTP/1.1"

Headers

  • Key/value pairs
  • Syntax: Key: Value
  • Can be anything
  • Ends with an empty line

Request body

For sending extra data with request, like

  • form data
  • files uploads
  • etc.

Another example

POST /order/form.html HTTP/1.0
User-Agent: SomeBrowser 5.8
Content-Type: application/x-www-form-urlencoded
Content-Length: 44

name=reindert&product=OCA_Book&amount=1

HTTP Response

Response Anatomy

Very similar to request.

  • Statuscode
  • Headers
  • Empty Line
  • Response body

Example

HTTP/1.1 200 OK
Date: Mon, 27 Jul 2009 12:28:53 GMT
Server: Apache/2.2.14 (Win32)
Last-Modified: Wed, 22 Jul 2009 19:15:56 GMT
Content-Length: 88
Content-Type: text/html
Connection: Closed

<html>
<body>
<h1>Hello, World!</h1>
</body>
</html>

Statuscodes

https://httpstatuses.com/

  • 1xx Informational
  • 2xx Success
  • 3xx Redirection
  • 4xx Client Error
  • 5xx Server Error

Headers

contain many kinds of information, including:

  • kind of data in body (MIME type)
  • data size
  • date resource was served
  • caching info

Body

contains the "payload": html, css, js, audio, video, etc. etc.
