Teamwork met Git

Conflicts

When working on the same files at the same time, you will run into conflicts.

Hands-on conflict

Let's try this:

  • teammate 1 makes a change to a file and commits (no push)
  • teammate 2 makes a change to the same line in the same file and pushes
  • teammate 1 pulls

Conflict

Code with a conflict looks like this 

<<<<<<< HEAD
    print("this is my code");
    return 0;
=======
    print("This is my coworkers code")
    return -1;
>>>>>>> branch 'master' of git@bitbucket.org:rjekker/git-test.git

Solve conflict

  • It's just text; change it until it's the way you want it
  • Or use a specialised merge tool
  • Then: commit
  • This is called a merge

Branches

We don't want to have to solve conflicts all the time

Usually everybody works on their own branch

You only merge branches when you want to.

Demo: local branch and merging!

Merge branches

  1. check out the branch you want to merge into
  2. Then merge with the branch with changes
  3. Solve conflicts
  4. Commit!
